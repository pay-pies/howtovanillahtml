What is HTML & CSS?

HTML stand for Hypertext Markup Language and CSS stands for Cascading Style Sheets. HTML is used to structure a web page and its contents. CSS is used to add style and describe the presentation of a markup language like HTML. The HTML will go in the body tag and CSS will go in the style in the head.

Vanilla HTML/CSS is the plain language without any extra plugins or fancy frameworks.

Text tags

There are 6 'h' sizes: h1, h2, h3 ... h6. h1 being the largest and h6 being the smallest. h1 is typically used for titles, h2 for headings, h3 for sub-heading etc. There is also 'p', which is your normal paragraph text.

There are 5 text tags you need to know: br, em, u, b and strong. The br tag is the line break tag. You will only need one of these unlike most tags. 'em' is the italic tag, which makes your text italic (wow! i know right). 'u' is the underline tag and b AND strong are bold text tags I guess. The strong tag implies that that piece of text is important but the b tag has no significant importance and is only for visuals.

Images

Within the img tag you can adjust the height and width values of the image. For these, you just type 'height=" [value] "' or 'width=" [value] ' inside.

Ids & Classes

If you wanted to style a certain piece text/image or something or group of things individually, you could give it an id or class. The difference between them is that an id can only be used by one element and a class can be used by multiple elements.

To set an id or class to something, go inside the tag and type 'id=""' or 'class=""' and type in the id/class name inside the quotation marks.

The CSS syntax for ids and classes are different. For ids you put an hashtag '#' and then the name and for classes you put a dot '.' and the name.

Links

You can have internal links and external links. For external links, you will use the a tag. Inside the a tag you type in 'href=" [url] "' and copy the url into the space where it says url.

You may notice links are always blue or purple and underlined. You can change the color with the CSS from before, but to remove the underline you can use the text-decoration property.

In CSS, set text-decoration to none and the underline will vanish. I don't think you will need an example for this as it is quite straightforward I think.

For internal links, you can add an id to the tag you want to go to when you press the link. Also using the a tag, you insert the id name with a hashtag '#' in front instead of the link inside the 'href=""'.

By the way, href stands for hypertext reference :).

There are five states of a link: link (default), hover, visited, active (when clicked), focus (when clicked/given focus).

We can change the properties when a list is in one of these states by typing 'a: [state] ' and then the CSS.

Comments

In a lot of languages, we are able to make comments. In HTML, we can use them to give ourselves/other people who are looking at our code extra information that you might not wan to show on the actual web page. In other cases, you might be experiments and wanting to comment out a line or two to see what happens.

The syntax for comments in HTML is '<! -- [comment] -->'.

BUT in CSS the syntax is different. Instead, you type in '/* [comment] */'.

Note: In both HTML and CSS, comments can go over multiple lines with the same syntax as single-line comments.

More CSS properties

To change the font of the text, you use the font-family property. There are 5 types of fonts in HTML: sans-serif, serif, monospace, cursive and fantasy. Examples of each: verdana, arial; times new roman, georgia; courier new, lucida; brush script mt, bradley hand and luminari.

MAKE SURE TO HAVE QUOTES AROUND THE FONT NAME

Some other properties include: text-decoration, text-align, border, font-style, line-height (line spacing), font-weight (light, normal, bold), width, height, overflow.

You can just style them as div and span or give them classes/ids.

The Box Model

This is the box model, consisting of the actual content, the padding, the border and the margin.

  • The content is your text and images.
  • The padding is the transparent area cleared around the content. This controls the space within an element.
  • The border is, well a border that goes around the content and padding.
  • The margin is the transparent area cleared around the border that seperates it from other elements.

We can modify the padding, border and margin using CSS with the padding, border and margin properties. If you wanted them on a specific side then you could use the padding-left/right/top/bottom, border-left/right/top/bottom etc.

For the values of the properties we use pixels, you just type in the number and 'px'.

CSS Position

The position property has 5 possible values: static (default), absolute, relative, fixed and sticky.

What they are:

  • static - the elements are rendered in order, as they appear in a document.
  • absolute - positioned relative to the first ancestor element (like how body is the ancestor element of everything in it)
  • relative - positioned relative to its normal position
  • fixed - positioned relative to the browser window
  • sticky - positioned based on the user's scrolling position

There is also initial and inherit. Initial resets it to its normal position and inherit makes the element inherit the property from its parent element.

There are right, left, top and bottom properties, which contain how many extra pixels you want to the right/light/top/bottom of the element.

Floating properties

The float property has two possible values: left and right. This will the make the element 'float' in that direction.

The clear property controls the flow next to floated elements. The possible values are: left, right, both, initial and inherit. 'both' refers to both left and right. left and right are pretty self-explanatory. We've mentioned intial and inherit before but I'll explain them again in case you've forgotten. Initial resets the property to its original value and inherit makes the element 'inherit' the property from its parent element.

Flexbox

The flexbox (flexible box layout) a layout method to organise items into rows and columns. A flexbox always has a flex container with the flex items inside. The container will be created using a div class.

The properties used for the flex container are: flex-direction, flex-wrap, flex-flow, justify-content, align-items and align-content.

The flex-direction can have one of these 4 values: row (default/horizontally), column (vertically), row-reverse (right to left), column-reverse (bottom to top).

The flex-wrap can have one of these values: nowrap(default), wrap(will wrap if necessary), wrap-reverse(same as wrap but in reverse).

The flex flow property is the short-hand version of changing the flex-direction and flex-wrap properties.

The justify-content property can have one of these values: center, flex-start (default), flex-end (places at end of container), space-around (places with space around each item), space-between (places with space between them), space-evenly (places items with equal space around them).

The align-items property can have one of these: center, flex-start (places at top), flex-end (places at bottom), stretch (same as normal), baseline (places at baseline), normal.

The align-content property has the same possible values but instead of aligning the items, it aligns the flex lines.

The display property has quite a few possible values so you can look at them over at w3schools (again).
